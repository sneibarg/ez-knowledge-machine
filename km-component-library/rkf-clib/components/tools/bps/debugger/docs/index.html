A Simple Demonstration of the Basic Problem Solver Debugger

Introduction

The Basic Problem Solver is acccompanied by a tool to support debugging the problem solving process. The debugging tool is an interactive command loop allowing a user to interact with the problem solver to better understand the problem solving process.

The debugger is invoked in the LISP context.

KM: (ps-debug)
      ---------------------------------------------
      A debugger for the Basic Problem Solver v0.01
      ---------------------------------------------
(ps-debug)

This note demonstrates the debugger  using the CPL output for the following 15July05 Physics demo question. 
An object is thrown from the top of a cliff.
The initial horizontal velocity of the object is 20 m/s.
The initial vertical velocity of the object is 0 m/s.
The height of the cliff is 125 m.
The object falls from the top of the cliff to the level ground.
What is the duration of the fall?
The CPL output for version 1.33b is available in question.triples .

The following two snippets demonstrate the basic features of the debugger.
  • Snippet #1 demonstrates an interaction capability allowing the user to crank the problem solver in a stepwise manner.
  • Snippet #2 demonstrates the adhoc mode which allows fine-grained interaction between the user and the problem solver.
A section briefly describing other commands supported by the debugger tool is also included.

Snippet #1

Demonstrates the interaction capability allowing the user to crank the problem solver in a stepwise manner.

Commands used in demo.

Command
 Remarks
begin <scenario> <question>
Initializes the problem solver context and loads up the necessary CPL outputs for the question.
The <scenario> and <question> arguments refer to the LISP objects created by CPL.
next
 Attempts problem solving on next Viewpoint and reverts control back to user.
continue
 Revert control to problem solver to investigate all Viewpoints without further interaction from user.
ls
 Returns a listing of the Viewpoints investigated by the problem solver.

Snippet #2

Demonstrates the adhoc mode which allows fine-grained interaction between the user and the problem solver.

Commands use in demo.

Command
 Remarks
begin <scenario> <question>
 Initializes the problem solver context and loads up the necessary CPL outputs for the question.
The <scenario> and <question> arguments refer to the LISP objects created by CPL.
adhoc
 For entering and exiting adhoc mode.
try <concept1> <concept2> ...
 Query problem solver if given concepts can be used as potential Viewpoints.
try all
 Queries problem solver for all potential Viewpoints.
pick
 User selects a potential Viewpoint to attempt problem solving.
unpick
 User unselects current Viewpoint and returns to previous Viewpoint to attempt problem solving.
why
 Queries problem solver on circumstances for current Viewpoint.
